# 学习目标

完成本课后，应能区分视频生成模型、交互环境模型、潜在动力学和控制世界模型；解释 UniSim、Genie、V-JEPA 2、Cosmos 与动作条件机器人视频模型的技术位置；设计检验模型是否真的理解动作后果的干预实验。

# 1. 世界模型必须对动作敏感

$$p_\theta(o_{t+1:t+H}\mid o_{\le t},a_{t:t+H-1},g)$$

> **读法：** 给定截至当前的观测历史、未来 H 步动作和任务条件，模型预测未来 H 步观测的概率分布。

**推导：** 环境的未来不仅取决于当前画面，也取决于智能体接下来施加的动作。把动作从条件中去掉，模型学习的是自然视频延续；把动作放入条件，才有可能回答反事实问题“如果动作换掉，未来会怎样变化”。

# 2. 四种模型不要混为一谈

| 模型 | 主要学习对象 | 能否直接用于控制 |
|-|-|-|
| 无条件/文本视频生成 | 视觉上合理的视频分布 | 通常不能 |
| 交互世界生成 | 输入动作或控制下的视频变化 | 需验证动作语义和时延 |
| 潜在动力学 | 控制相关状态的转移 | 可用于规划或想象学习 |
| 控制世界模型 | 未来状态、奖励、约束与不确定性 | 目标就是改进决策 |

# 3. 像素似然不等于物理正确

视频生成常最小化负对数似然或其变分上界：

$$\mathcal L_{video}=-\mathbb E_{(o,a)\sim\mathcal D}\log p_\theta(o_{t+1:t+H}\mid o_{\le t},a)$$

> **读法：** 在真实视频和动作数据上，让模型给实际发生的未来画面分配更高概率。

**推导：** 最大似然等价于最小化经验负对数似然。但视觉损失会同时惩罚纹理、光照和背景误差；一个在像素上更准的模型，未必更准确地保留接触、物体身份和可达性。

# 4. 潜在预测：只保留决策需要的信息

$$z_t=f_\theta(o_{\le t}),\qquad \hat z_{t+1}=F_\phi(z_t,a_t)$$

> **读法：** 编码器把观测历史压缩成潜在状态，动力学模型根据当前潜在状态和动作预测下一潜在状态。

**推导：** 若原始观测含有大量与控制无关的像素，先构造状态表示再预测，可把容量集中到对象、运动和交互结构。但潜在空间必须通过奖励、动作、对比或多步预测等监督，避免丢掉控制相关变量。

# 5. UniSim：从真实视频学习可交互模拟器

UniSim 代表“生成式真实世界模拟器”路线：使用视觉数据与动作条件生成可交互未来。评审重点不是单帧逼真度，而是同一动作是否可重复地产生一致后果、替换动作是否产生正确反事实、长时 rollout 是否保持对象身份和几何关系。

# 6. Genie 与 Genie 2：从视频发现可控制环境

Genie 展示从互联网视频学习潜在动作和可交互环境的路线；Genie 2 进一步强调大规模生成式世界模型。对 Physical AI 的价值在于无需现成机器人动作标签也可能发现“可控制变化”，但潜在动作与真实机器人力矩之间仍存在 embodiment grounding 问题。

# 7. V-JEPA 2 与动作条件规划

JEPA 路线不要求重建每个像素，而是在表示空间预测被遮挡或未来内容。一个抽象目标是：

$$\mathcal L_{JEPA}=\|\hat z_{future}-\operatorname{sg}(z_{future})\|_2^2$$

> **读法：** 让上下文预测器输出的未来表示接近目标编码器产生的真实未来表示；停止梯度算子使目标分支不被这一项直接更新。

**推导：** 用目标编码器构造预测目标，可以跳过像素级细节。V-JEPA 2 展示大规模视频自监督表示，并通过动作条件模型连接机器人规划。关键实验是：固定控制数据量后，视频预训练是否提高动作后果预测和真实任务成功率。

# 8. Cosmos：世界基础模型作为数据与仿真基础设施

Cosmos 强调面向 Physical AI 的世界生成与数据基础设施，包括视频生成、条件控制和可定制模型。课程将其视为“世界模型平台”而非单一控制算法：生成数据是否提升策略，取决于动作条件是否正确、场景覆盖是否匹配以及合成误差是否被策略利用。

# 9. 动作条件机器人视频模型

机器人模型常预测未来图像、对象轨迹或视觉子目标，再用逆动力学或规划器得到动作。候选动作序列可用预测回报评分：

$$\hat G(a_{t:t+H-1})=\sum_{k=0}^{H-1}\gamma^k\hat r(\hat z_{t+k},a_{t+k})+\gamma^H\hat V(\hat z_{t+H})$$

> **读法：** 一个候选动作序列的预测总收益，等于模型 rollout 中每一步预测奖励的折扣和，再加上终点状态的折扣价值。

**推导：** 这是有限时域回报定义。世界模型产生未来潜在状态，奖励和价值模型把状态转成决策分数，规划器比较多条候选序列。

# 10. 论文事实与证据边界

| 路线 | 公开事实 | 作者解释 | 课程判断 |
|-|-|-|-|
| UniSim | 用真实世界视频和动作条件训练生成式交互模拟器，并展示策略训练与评测用途 | 生成模型可扩大机器人和智能体可交互经验 | 关键不是单帧真实感，而是动作语义、长时一致性和模拟器到真实系统的收益 |
| Genie / Genie 2 | 从视频学习潜在动作或可交互生成环境，强调大规模世界生成能力 | 没有显式动作标签的视频也包含可控制结构 | 潜在动作仍需与机器人本体和控制频率 grounding，不能直接等同于机器人动作 |
| V-JEPA 2 | 通过视频自监督表征与动作条件模型连接预测和机器人规划 | 表示空间预测可跳过不必要的像素生成 | 应固定机器人控制数据量，检验预训练是否真正提高动作后果预测和真实任务成功率 |
| Cosmos | 提供面向 Physical AI 的世界生成模型、数据处理与可定制基础设施 | 世界基础模型可以支持合成数据、仿真和下游 Physical AI 开发 | 它是平台路线而非单一控制算法；合成数据必须用真实闭环收益和偏差审计证明价值 |

# 11. 失败模式：三类决定性误差

| 误差 | 现象 | 控制后果 |
|-|-|-|
| 动作不敏感 | 换动作后预测几乎不变 | 无法做反事实规划 |
| 接触不一致 | 穿透、漂浮、物体瞬移 | 错误估计可达性和风险 |
| 多步漂移 | 短期逼真、长期身份丢失 | 长时规划目标失真 |

# 12. 最小实验

1. 构造相同起始画面、两个相反动作，检查预测未来是否显著且正确地分叉。
2. 分别测像素误差、对象位置误差、接触事件误差和规划成功率。
3. 随着 rollout horizon 增长画误差曲线，定位控制收益开始下降的位置。
4. 对背景、纹理和光照做变化，验证潜在模型是否保留物理状态而忽略无关像素。
5. 固定策略数据量，比较无视频预训练、视频生成预训练与 JEPA 预训练。

# 13. 本课练习

1. 解释为什么高 PSNR 不能证明世界模型适合控制。
2. 设计一个检验模型是否利用动作条件的置换实验。
3. 比较像素生成、潜在预测和奖励预测各自可能丢失的信息。
4. 说明潜在动作从人类视频迁移到机器人时需要哪些 grounding。
5. 为接触丰富任务定义一个比视频逼真度更关键的世界模型指标。
