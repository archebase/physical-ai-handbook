# 学习目标

完成本课后，应能区分视频生成模型、交互环境模型、潜在动力学、World Action Model 与控制世界模型；解释 UniSim、Genie、V-JEPA 2、Cosmos、WAM4D 与 X-WAM 的技术位置；设计检验模型是否真的理解动作后果、三维几何和接触约束的干预实验。

# 1. 世界模型必须对动作敏感

$$p_\theta(o_{t+1:t+H}\mid o_{\le t},a_{t:t+H-1},g)$$

> **读法：** 给定截至当前的观测历史、未来 H 步动作和任务条件，模型预测未来 H 步观测的概率分布。

**推导：** 环境的未来不仅取决于当前画面，也取决于智能体接下来施加的动作。把动作从条件中去掉，模型学习的是自然视频延续；把动作放入条件，才有可能回答反事实问题“如果动作换掉，未来会怎样变化”。

# 2. 四种模型不要混为一谈

| 模型 | 主要学习对象 | 能否直接用于控制 |
|-|-|-|
| 无条件/文本视频生成 | 视觉上合理的视频分布 | 通常不能 |
| 交互世界生成 | 输入动作或控制下的视频变化 | 需验证动作语义和时延 |
| 潜在动力学 | 控制相关状态的转移 | 可用于规划或想象学习 |
| 控制世界模型 | 未来状态、奖励、约束与不确定性 | 目标就是改进决策 |

# 3. 像素似然不等于物理正确

视频生成常最小化负对数似然或其变分上界：

$$\mathcal L_{video}=-\mathbb E_{(o,a)\sim\mathcal D}\log p_\theta(o_{t+1:t+H}\mid o_{\le t},a)$$

> **读法：** 在真实视频和动作数据上，让模型给实际发生的未来画面分配更高概率。

**推导：** 最大似然等价于最小化经验负对数似然。但视觉损失会同时惩罚纹理、光照和背景误差；一个在像素上更准的模型，未必更准确地保留接触、物体身份和可达性。

# 4. 潜在预测：只保留决策需要的信息

$$z_t=f_\theta(o_{\le t}),\qquad \hat z_{t+1}=F_\phi(z_t,a_t)$$

> **读法：** 编码器把观测历史压缩成潜在状态，动力学模型根据当前潜在状态和动作预测下一潜在状态。

**推导：** 若原始观测含有大量与控制无关的像素，先构造状态表示再预测，可把容量集中到对象、运动和交互结构。但潜在空间必须通过奖励、动作、对比或多步预测等监督，避免丢掉控制相关变量。

# 5. UniSim：从真实视频学习可交互模拟器

UniSim 代表“生成式真实世界模拟器”路线：使用视觉数据与动作条件生成可交互未来。评审重点不是单帧逼真度，而是同一动作是否可重复地产生一致后果、替换动作是否产生正确反事实、长时 rollout 是否保持对象身份和几何关系。

# 6. Genie 与 Genie 2：从视频发现可控制环境

Genie 展示从互联网视频学习潜在动作和可交互环境的路线；Genie 2 进一步强调大规模生成式世界模型。对 Physical AI 的价值在于无需现成机器人动作标签也可能发现“可控制变化”，但潜在动作与真实机器人力矩之间仍存在 embodiment grounding 问题。

# 7. V-JEPA 2 与动作条件规划

JEPA 路线不要求重建每个像素，而是在表示空间预测被遮挡或未来内容。一个抽象目标是：

$$\mathcal L_{JEPA}=\|\hat z_{future}-\operatorname{sg}(z_{future})\|_2^2$$

> **读法：** 让上下文预测器输出的未来表示接近目标编码器产生的真实未来表示；停止梯度算子使目标分支不被这一项直接更新。

**推导：** 用目标编码器构造预测目标，可以跳过像素级细节。V-JEPA 2 展示大规模视频自监督表示，并通过动作条件模型连接机器人规划。关键实验是：固定控制数据量后，视频预训练是否提高动作后果预测和真实任务成功率。

# 8. Cosmos：世界基础模型作为数据与仿真基础设施

Cosmos 强调面向 Physical AI 的世界生成与数据基础设施，包括视频生成、条件控制和可定制模型。课程将其视为“世界模型平台”而非单一控制算法：生成数据是否提升策略，取决于动作条件是否正确、场景覆盖是否匹配以及合成误差是否被策略利用。

# 9. 动作条件机器人视频模型

机器人模型常预测未来图像、对象轨迹或视觉子目标，再用逆动力学或规划器得到动作。候选动作序列可用预测回报评分：

$$\hat G(a_{t:t+H-1})=\sum_{k=0}^{H-1}\gamma^k\hat r(\hat z_{t+k},a_{t+k})+\gamma^H\hat V(\hat z_{t+H})$$

> **读法：** 一个候选动作序列的预测总收益，等于模型 rollout 中每一步预测奖励的折扣和，再加上终点状态的折扣价值。

**推导：** 这是有限时域回报定义。世界模型产生未来潜在状态，奖励和价值模型把状态转成决策分数，规划器比较多条候选序列。

## 9.1 4D World Action Model：从视频未来到时变三维世界

World Action Model（WAM）把未来观测与机器人动作放进同一个生成或预测模型，使视觉未来和可执行动作共享时序表示。4D-WAM 在此基础上显式引入随时间变化的三维结构。这里的“4D”指三维空间加时间，而不是第四个空间维度，也不等于模型已经学到完整刚体动力学、摩擦、接触力或材料顺应性。

### 9.1.1 4D 表示究竟增加了什么

普通视频模型输出二维像素序列。它可以生成视觉上合理的抓取，却可能把机械手画在物体内部、改变物体尺度，或在遮挡后丢失对象身份。4D 表示要求不同时刻的视觉结果能够落到一致的三维坐标关系中，从而减少单目尺度歧义、跨视角不一致和遮挡区域的错误补全。

| 表示 | 保留的信息 | 主要代价或缺口 |
|-|-|-|
| 多视角 RGB-D 序列 | 每个时刻的颜色、深度和跨视角几何 | 依赖相机标定；深度不直接给出接触力 |
| 点云或 point map 序列 | 对象表面、空间距离和三维运动 | 遮挡区域稀疏；对应关系和长时身份容易漂移 |
| Scene flow / 三维轨迹 | 三维点随时间的位移 | 难以处理拓扑变化、接触切换和不可见表面 |
| 动态 NeRF / Gaussian 表示 | 连续视角渲染和动态场景外观 | 优化与渲染成本高，未必适合低延迟动作生成 |
| 几何 latent / register token | 把空间监督压入控制相关隐变量 | 解释性较弱，必须证明 latent 确实改善控制 |

**关键区别。** 4D 世界生成强调“未来世界在三维中是否一致”；机器人控制还要求“动作在当前本体、接触和时延约束下是否可执行”。前者是后者的重要条件，但不是充分条件。

### 9.1.2 WAM 联合建模什么

设历史多视角观测为 $O_t^{hist}$，未来深度或几何为 $G_{t+1:t+H}$，历史动作为 $a_{t-L:t-1}$，任务条件为 $c$。统一的 4D World Action Model 可以写成：

$$p_\theta(O_{t+1:t+H},G_{t+1:t+H},a_{t:t+K-1}\mid O_t^{hist},a_{t-L:t-1},c)$$

> **读法：** 模型根据历史视觉、历史动作和任务条件，联合预测未来视觉、未来三维几何与接下来要执行的动作块。

这与“先用世界模型预测未来，再由独立规划器搜索动作”并不完全相同。WAM 可以联合去噪或联合生成视觉与动作，动作头直接从共享表示中得到动作块；未来视频和几何既可以作为显式输出，也可以只作为辅助训练目标。因而判断一个 WAM 是否真有世界建模能力，不能只看动作成功率；判断它是否真能控制，也不能只看视频、深度或点云质量。

### 9.1.3 WAM4D：用 Spatial Register 蒸馏几何先验

WAM4D 建立在因果视频—动作模型之上。决策时的因果上下文包含语言、多视角 RGB 历史与历史动作：

$$C_t=\{l,O_t^{hist},a_{t-L_a:t-1}\}$$

训练时，未来视频 latent 与未来动作都被加噪并放入 Mixture-of-Transformers 序列：

$$X_t^{(0)}=[Z_t^{hist},\widetilde Z_t^{fut},A_t^{hist},\widetilde A_t^{fut}]$$

模型分别预测未来视频和动作的 flow target。WAM4D 的新增机制不是在推理时生成完整点云，而是在训练期加入一组可学习的 spatial register。register 按未来深度时刻和多视角拼图位置复制，以 register 为 query，从中层历史视频特征读取空间信息：

$$R_t^{\ell+1}=\operatorname{DepthBlock}_\ell(Q=R_t^\ell,K,V=[R_t^\ell,Z_t^{hist,\ell}])$$

更新后的 register 被送入由 Depth Anything 3 初始化的几何头，预测未来深度。默认设计在主干第 12、14、16、18 层接入深度读取块，使几何损失反向更新动作预测所依赖的历史视觉特征。论文默认的三视角、八个未来深度帧共产生 960 个 register token；这些 token 是训练期几何查询，不是部署时的额外传感器输入。

**因果可见性是核心。** 如果未来动作能够读取真实未来视频或未来深度，模型会通过训练期捷径“偷看答案”，部署时性能将崩溃。WAM4D 使用 causal mixture attention 限制各模态的可见范围：

| 查询 token | 允许读取 | 禁止读取 |
|-|-|-|
| 未来动作 | 历史视频、历史动作、自身加噪动作 token | 未来视频 token、spatial register |
| 未来视频 | 合法历史上下文及视频去噪状态 | 不能作为未来动作的真值条件 |
| Spatial register | 自身与历史视频特征 | 未来视频真值、动作策略路径 |

总损失由视频、动作和深度三部分组成：

$$\mathcal L=\mathcal L_{video}+\lambda_{act}\mathcal L_{action}+\lambda_{depth}\mathcal L_{depth}$$

其中深度项对有效像素使用 SmoothL1。部署时移除 register、深度块与几何头，仅保留观测到动作的因果路径和 KV-cache。因此 WAM4D 更准确的描述是“用未来深度监督蒸馏空间先验的轻量动作模型”，而不是“部署时持续显式生成 4D 世界的模拟器”。

**证据边界。** 论文在 RoboTwin 十任务消融中报告：无深度版本 Clean SR 为 71.7，中层单向 register 为 75.2；双向 register 达到 76.6，但会让主干读取几何 token，增加计算与复杂度，且多数几何指标反而下降。真实数据的深度监督来自离线 Depth Anything 3 伪深度，因此结果同时包含几何教师质量这一混杂变量，不能等同于真实传感器深度监督。

### 9.1.4 X-WAM：统一 RGB-D、状态、动作与异步去噪

X-WAM 从预训练视频扩散 Transformer 出发，以语言、初始多视角 RGB 和初始本体状态为条件，联合预测未来 RGB、深度、本体状态和动作：

$$Z=[z_{O_0},z_{O_{1:H}},z_{s_0},z_{s_{1:H}},z_{a_{1:K}}]$$

论文配置预测 8 个未来 RGB/状态时刻和 32 个动作，动作频率是视频频率的四倍。多视角 token 通过 view embedding 区分相机；静态相机使用固定外参，腕部相机位姿由预测末端位姿和手眼标定矩阵得到：

$$T_{wrist}=T_{ee}T_{h2e}$$

这样各视角的 RGB-D 才能融合到统一三维坐标系。该设计同时意味着相机外参、手眼标定或末端位姿预测的误差会直接污染 4D 重建。

**轻量深度分支。** X-WAM 不把深度 token 直接拼到主序列，因为序列长度和注意力成本会显著增加；也不把 RGB 与深度简单做通道拼接，因为这会偏离预训练视频模型的输入分布。它复制预训练 DiT 最后的若干 block 形成深度分支。深度分支通过 unilateral attention 读取主分支中间特征，但主分支不读取深度 token，从而尽量保留视频预训练权重和推理效率。深度分支回归 inverse depth，并可在动作解码时关闭。

**Asynchronous Noise Sampling。** 视频通常需要较多去噪步数才能获得清晰未来，低维动作则可以更快解码。设动作/状态需要 $T_a$ 步，视频需要 $T_O$ 步且 $T_a\lt T_O$。推理到第 $T_a$ 步时即可把干净动作发送给机器人，视频分支继续去噪；此后的世界生成自然变成以已解码动作为条件的预测。

如果训练时独立随机采样视频噪声 $t_O$ 与动作噪声 $t_a$，会产生推理时不会出现的状态，例如视频比动作更干净。ANS 改为从满足 $t_O\ge t_a$ 的联合分布采样，并显式加入“动作已经干净、视频仍在去噪”的训练情形，使训练分布覆盖异步推理路径。其总目标为：

$$\mathcal L_{total}=\mathcal L_O+\lambda_s\mathcal L_s+\lambda_a\mathcal L_a+\lambda_D\mathcal L_{depth}$$

**论文报告的消融。** 在 RoboCasa 消融设置中，无深度分支的成功率为 63.0%，交错深度分支为 67.8%；显式序列拼接得到 68.7%，但动作延迟从 1033 ms 增加到 1888 ms。同步 25 步去噪延迟为 4665 ms，ANS 异步推理为 1033 ms。上述数字证明了论文设计中的质量—延迟权衡，但仍不能直接外推到不同 GPU、控制频率或真实机器人。

### 9.1.5 WAM4D 与 X-WAM 的差异

| 维度 | WAM4D | X-WAM |
|-|-|-|
| 4D 几何角色 | 训练期辅助蒸馏目标 | 联合生成与重建输出 |
| 几何接口 | Spatial register 查询历史视觉特征 | 复制末端 DiT block 的交错深度分支 |
| 动作因果性 | 未来动作禁止读取未来视频和 register | 多模态联合去噪，动作可先完成并条件化后续视频 |
| 部署几何成本 | 几何分支完全移除 | 深度分支可关闭或降低运行频率 |
| 主要技术矛盾 | 怎样让几何监督进入策略而不增加推理成本 | 怎样同时保持 4D 生成质量和实时动作速度 |
| 主要审计风险 | 伪深度教师、训练期捷径、几何监督是否真改善动作 | 大规模预训练混杂、标定误差、异步训练—推理一致性 |

### 9.1.6 4D 几何不等于完整物理

RGB-D、点云或 scene flow 主要描述运动学几何。它们可以告诉模型“物体在哪里、表面怎样移动”，却通常不能唯一确定摩擦系数、质量分布、接触刚度、夹持力、执行器饱和和控制延迟。同一段视觉运动可能由不同的力与材料属性产生；没有力觉、本体反馈或干预数据时，这些变量往往不可辨识。

| 决定性实验 | 控制变量 | 要否定的弱解释 |
|-|-|-|
| 移除或打乱深度监督 | 固定动作数据、主干参数量和训练计算 | 收益只是更多参数或正则化 |
| 单视角与多视角比较 | 固定帧数与像素预算 | 收益只是输入数据量增加 |
| 真实深度与伪深度比较 | 相同任务与策略训练流程 | 模型学到的是教师模型偏差 |
| 标定扰动实验 | 扫描相机外参、手眼标定和时间同步误差 | 4D 表示对部署标定天然鲁棒 |
| 几何—控制相关性 | 按任务报告深度误差、点云误差和成功率 | 更好几何指标必然带来更好控制 |
| 接触与物理 OOD | 改变摩擦、质量、顺应性、透明反光和遮挡 | 视觉空间一致性等于物理理解 |
| 延迟—成功率曲线 | 统一硬件、去噪步数、动作块长度和控制频率 | 离线动作精度足以证明实时可用 |

**课程判断。** 4D-WAM 是视频世界模型走向机器人控制的重要升级，因为它把空间结构、跨视角一致性与动作生成放进同一证据链。但其最强结论应限制为：几何监督可以改善控制相关视觉表示，并可能提高动作成功率；只有当模型进一步通过力觉、本体反馈、真实干预和闭环恢复验证后，才有资格声称学到了可用于接触丰富任务的物理世界模型。

# 10. 论文事实与证据边界

| 路线 | 公开事实 | 作者解释 | 课程判断 |
|-|-|-|-|
| UniSim | 用真实世界视频和动作条件训练生成式交互模拟器，并展示策略训练与评测用途 | 生成模型可扩大机器人和智能体可交互经验 | 关键不是单帧真实感，而是动作语义、长时一致性和模拟器到真实系统的收益 |
| Genie / Genie 2 | 从视频学习潜在动作或可交互生成环境，强调大规模世界生成能力 | 没有显式动作标签的视频也包含可控制结构 | 潜在动作仍需与机器人本体和控制频率 grounding，不能直接等同于机器人动作 |
| V-JEPA 2 | 通过视频自监督表征与动作条件模型连接预测和机器人规划 | 表示空间预测可跳过不必要的像素生成 | 应固定机器人控制数据量，检验预训练是否真正提高动作后果预测和真实任务成功率 |
| Cosmos | 提供面向 Physical AI 的世界生成模型、数据处理与可定制基础设施 | 世界基础模型可以支持合成数据、仿真和下游 Physical AI 开发 | 它是平台路线而非单一控制算法；合成数据必须用真实闭环收益和偏差审计证明价值 |
| WAM4D | 用训练期 spatial register tokens 引入未来深度监督，并在动作推理时移除几何读取分支 | 可把预训练几何先验注入因果视频—动作模型，同时避免稠密 4D 解码拖慢控制 | 需证明几何监督在固定数据和计算预算下改善真实闭环，而不只是改善深度或仿真指标 |
| X-WAM | 联合预测多视角 RGB-D 未来与机器人动作，并用异步去噪平衡动作速度和世界生成质量 | 统一 4D 世界建模与实时动作执行可以兼顾空间理解和控制效率 | 应拆分预训练数据规模、几何分支和异步采样的贡献，并审计接触、延迟与跨本体泛化 |

# 11. 失败模式：三类决定性误差

| 误差 | 现象 | 控制后果 |
|-|-|-|
| 动作不敏感 | 换动作后预测几乎不变 | 无法做反事实规划 |
| 接触不一致 | 穿透、漂浮、物体瞬移 | 错误估计可达性和风险 |
| 多步漂移 | 短期逼真、长期身份丢失 | 长时规划目标失真 |

# 12. 最小实验

1. 构造相同起始画面、两个相反动作，检查预测未来是否显著且正确地分叉。
2. 分别测像素误差、对象位置误差、接触事件误差和规划成功率。
3. 随着 rollout horizon 增长画误差曲线，定位控制收益开始下降的位置。
4. 对背景、纹理和光照做变化，验证潜在模型是否保留物理状态而忽略无关像素。
5. 固定策略数据量，比较无视频预训练、视频生成预训练与 JEPA 预训练。

# 13. 本课练习

1. 解释为什么高 PSNR 不能证明世界模型适合控制。
2. 设计一个检验模型是否利用动作条件的置换实验。
3. 比较像素生成、潜在预测和奖励预测各自可能丢失的信息。
4. 说明潜在动作从人类视频迁移到机器人时需要哪些 grounding。
5. 为接触丰富任务定义一个比视频逼真度更关键的世界模型指标。
