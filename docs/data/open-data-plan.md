# 免费数据开放计划

电子书和数据要同时公开，但两者的发布边界不同：正文可以先上线，机器人数据必须在授权、脱敏、质量和可复现证据齐全后再放出。

## 建议的 Starter Release

第一版不追求“小时数很大”，而追求可训练、可审计、可复现：

- 100–300 个 episode，覆盖 5–10 个明确任务；
- 每个 episode 同步保存观测、动作、状态、任务指令、时间戳和 embodiment 元数据；
- 至少保留一部分失败、接管和恢复片段，而不是只发布成功演示；
- 公开 train/validation/test 划分，按任务、对象或场景留出，避免随机帧泄漏；
- 同时提供原始索引和一种主流训练格式（例如 LeRobotDataset 或 Parquet manifest）；
- 每个版本附 `manifest.json`、SHA-256、采集设备、标定、脱敏记录和数据卡片。

这是一份执行目标，不是对尚未核验资产的数量承诺。实际数量应以采集、审查和授权结果为准。

## 分发层级

| 层级 | 放置位置 | 内容 |
|-|-|-|
| 代码与小样例 | GitHub 仓库 | Schema、读取器、10–50 MB 的脱敏样例、校验脚本 |
| 版本索引 | GitHub Release | manifest、数据卡片、版本说明、校验和 |
| 大文件 | Hugging Face / ModelScope / 对象存储 | 图像、视频、轨迹和分片压缩包 |
| 论文复现实验 | 仓库 CI 或 Colab | 只依赖小样例的最小实验，不要求下载完整数据 |

## 发布门禁

在任何“免费数据”链接公开前逐项签字：

1. 采集者、设备、场景和参与者授权可公开再分发。
2. 人脸、声音、车牌、客户标识、屏幕内容和文件路径已脱敏。
3. 第三方数据、机器人平台和背景音乐等许可证允许再分发。
4. 训练/验证/测试切分不存在 episode 或对象泄漏。
5. 随机读取、时间同步、动作单位和坐标系可由读取器验证。
6. 失败标签、缺失值和异常值有数据字典。
7. 数据集单独声明许可证；不要用代码许可证替代数据许可证。

## 当前仓库提供什么

当前先提供 [`episode.schema.json`](./schema/episode.schema.json) 和一个合成、无真实机器人隐私的 `downloads/sample-episodes.jsonl`（构建后可从站点根目录下载）。它们用于让读者现在就能跑通解析和校验；真实采集数据进入公开 Release 前，必须走上面的门禁。
